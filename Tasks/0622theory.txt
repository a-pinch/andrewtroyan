1. Наследование - процесс приобретения конкретным классом полей и методов (с учетом индентификаторов доступа: private, protected,
public) от базового(-ых) класса(-ов).
   Абстрактный класс - класс, у которого есть хотя бы один чисто абстрактный метод (virtual return_type method_name(parameters)
  = 0). Нельзя создавать экземпляры абстрактного класса!
   Таблица виртуальных методов - список адресов методов класса с идентификатором virtual. Если в классе есть хоть один виртуальный
   метод, то создается невидимое для нас поле - указатель на список адресов методов. При первой встрече виртуального метода, его
   адрес заносится в таблицу. И, если этот метод переопределяется, то адрес нового метода перезаписывается в таблице над 
   предыдущим.
   Поле - данные класса (переменные, структуры, контейнеры, экземпляры других классов и т.д.). Метод ПРЕДСТАВЛЕНИЯ класса.
   Список инициализации - блок действий в конструкторах между именем и телом. Помогает при произведении операций над полями
   класса перед вызовом их конструкторов.
   Экземпляр класса - копия класса с проинициализированными полями.
   Полиморфизм - пренебрежение каких-то особенностей выполнения методов экземпляров различных классов для достижения абстракции 
   (достигается при помощи виртуальных методов). Например, у конкретного класса A есть виртуальный метод virt(). От него 
   наследуются классы B и C, в которых этот метод переопределяется. Благодаря этому мы можем создать массив arr из 
   указателей (ссылок) на объект А, а в него занести указатели (ссылки) на классы B и C. И через элементы этого массива вызывать
   метод virt() (например, arr[1]->virt()). Достигается абстракция (нас не волнуют особенности выполнения данного метода).
   
2. ПОДКЛЮЧАТЬ необходимо файлы с расширением .h.
   H-файлы являются неким интерфейсом между кодом и тем, кто использует этот код. H-файлы нужны для того, чтобы при компиляции 
   программы код не вставлялся повторно. Для этого в h-файлах используются команды препроцессора (#pragma once или 
   #IFNDEF имя #DEFINE имя #ENDIF). 
   H-файлы обычно содержат в себе объявления классов, прототипы функций и т.д.
   А в cpp-файлах содержится непосредственно код (реализация методов, функций и т.д.).
